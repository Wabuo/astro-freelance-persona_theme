## Privacy & Licensing

This project takes contributor privacy and accurate licensing seriously. Please follow these guidelines:

### 1. Privacy-First Commits

We encourage all contributors to keep their personal email addresses private by using GitHub's `noreply` email feature.

**How to set it up:**

1. Go to your [GitHub Email Settings](https://github.com/settings/emails).
2. Check "Keep my email address private".
3. Note your `noreply` email address (usually `ID+username@users.noreply.github.com`).
4. Configure git for this repository:

    ```bash
    git config user.email "ID+username@users.noreply.github.com"
    ```

### 2. Licensing & Copyright

We use the [REUSE](https://reuse.software/) standard to ensure clear and machine-readable licensing.

- **Collective Copyright**: To protect privacy and maintain a unified project identity, we use a collective copyright notice for original code:
    `SPDX-FileCopyrightText: 2026 The freelance-persona_theme Project Contributors`
- **Do Not Add Personal Headers**: Please do not add your own name or email to file headers. Your contribution is recorded in the git history and acknowledged via our [All Contributors](#contributors) system in the README.
- **Attribution**: If you are porting code from another open-source project, please consult with the maintainers to ensure the original author is properly attributed in the file header alongside the collective project header.

### 3. All Contributors

We recognize every contribution, not just code! usage of the `@all-contributors` bot is encouraged to acknowledge your work.
